# 接口文档

**标准返回格式（JSON）**：

```json
{
  "code": 200,         // 状态码：200为成功，400/401/500等为失败
  "message": "success",// 提示信息，报错时前端可直接展示该信息
  "data": {}           // 具体的数据载荷
}
```

【功能介绍】：
- 认证模块支持邮箱验证码注册、登录与重置密码，JWT 令牌有效期 30 分钟，并写入 Redis 作为会话校验。
- 用户资料模块维护运势生成所需字段：生日、星座、性别、城市、职业。
- 运势模块按“用户 + 日期”查询缓存，未命中则调用大模型生成并落库。
- 订阅模块支持订阅/取消订阅，每日 07:30 批量生成并发送订阅用户运势，清理 7 天前历史数据。

以下是具体的 API 接口设计文档：

---

### 一、 认证与授权模块 (Auth)

#### 1. 发送邮箱验证码
合并了注册和重置密码的发送验证码逻辑，通过 `business_type` 进行区分。

* **接口地址**：`/api/v1/auth/send-code`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "business_type": 1  // 1-注册，2-重置密码
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "验证码发送成功",
      "data": null
    }
    ```

#### 2. 用户注册
* **接口地址**：`/api/v1/auth/register`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "password": "my_password123", // 前端可直接传明文
      "code": "123456"              // 邮箱验证码
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "注册成功",
      "data": null
    }
    ```

#### 3. 用户登录
* **接口地址**：`/api/v1/auth/login`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "password": "my_password123"
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "登录成功",
      "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", // JWT Token
        "expires_in": 1800 // 有效期 30 分钟 (单位: 秒)
      }
    }
    ```

#### 4. 重置密码
* **接口地址**：`/api/v1/auth/reset-password`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "new_password": "new_password123",
      "code": "123456"
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "密码重置成功",
      "data": null
    }
    ```

### 二、 运势业务模块 (Fortune)

*(注意：以下接口需要在 HTTP 请求头 Header 中携带 `Authorization: Bearer <Token>` 进行 JWT 鉴权)*

#### 5. 获取今日运势
对应功能 4.1。后端逻辑：
1) 先检查用户是否已完善资料（birthday/constellation/gender/city/occupation）；
2) 资料不完整则返回错误；
3) 查库 -> 有则直接返回 -> 无则调用大模型 API -> 存库 -> 返回。

* **接口地址**：`/api/v1/fortune/today`
* **请求方式**：`GET`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数**：无
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "success",
      "data": {
        "date": "2026-04-19",
        "content": "今日运势星级：★★★☆ ..."
      }
    }
    ```
* **资料缺失返回示例**：
    ```json
    {
      "code": 4003,
      "message": "请先完善运势资料: birthday,constellation",
      "data": null
    }
    ```

### 三、 用户设置模块 (User)

*(同样需要 JWT 鉴权)*

#### 6. 获取用户运势资料
* **接口地址**：`/api/v1/user/profile`
* **请求方式**：`GET`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数**：无
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "success",
      "data": {
        "user_id": 1,
        "birthday": "1999-08-12",
        "constellation": "狮子座",
        "gender": "男",
        "city": "上海",
        "occupation": "产品经理"
      }
    }
    ```

#### 7. 保存用户运势资料
* **接口地址**：`/api/v1/user/profile`
* **请求方式**：`PUT`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数** (Body - JSON)：
    ```json
    {
      "birthday": "1999-08-12",
      "constellation": "狮子座",
      "gender": "男",
      "city": "上海",
      "occupation": "产品经理"
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "用户资料已保存",
      "data": null
    }
    ```

#### 8. 订阅每日运势邮件
对应功能 4.2。

* **接口地址**：`/api/v1/user/subscribe`
* **请求方式**：`POST`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数**：无
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "已订阅，以后每日8点将发送当天运势给你",
      "data": null
    }
    ```

#### 9. 取消订阅每日运势邮件
对应功能 4.3。

* **接口地址**：`/api/v1/user/unsubscribe`
* **请求方式**：`POST`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数**：无
* **成功返回**：
    ```json
    {
      "code": 200,

      "data": null
    }
    ```

### 💡 工程师的开发小贴士：

1.  **关于 HTTP 状态码 vs 业务状态码**：我在这里采用了 HTTP 状态码一律返回 `200 OK`，具体的业务错误通过 JSON 里的 `code` 和 `message` 体现，这在前端统一拦截响应时更方便。
2.  **JWT 解析提取信息**：对于 `/fortune/today`、`/user/profile`、`/user/subscribe` 这类接口，前端不需要传 `user_id`，后端中间件应从 JWT payload 解析并写入上下文。
