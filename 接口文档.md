# 接口文档

**标准返回格式（JSON）**：

```json
{
  "code": 200,         // 状态码：200为成功，400/401/500等为失败
  "message": "success",// 提示信息，报错时前端可直接展示该信息
  "data": {}           // 具体的数据载荷
}
```

以下是具体的 API 接口设计文档：

---

### 一、 认证与授权模块 (Auth)

#### 1. 发送邮箱验证码
合并了注册和重置密码的发送验证码逻辑，通过 `business_type` 进行区分。

* **接口地址**：`/api/v1/auth/send-code`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "business_type": 1  // 1-注册，2-重置密码
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "验证码发送成功",
      "data": null
    }
    ```

#### 2. 用户注册
* **接口地址**：`/api/v1/auth/register`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "password": "my_password123", // 前端可直接传明文
      "code": "123456"              // 邮箱验证码
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "注册成功",
      "data": null
    }
    ```

#### 3. 用户登录
* **接口地址**：`/api/v1/auth/login`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "password": "my_password123"
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "登录成功",
      "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", // JWT Token
        "expires_in": 1800 // 有效期 30 分钟 (单位: 秒)
      }
    }
    ```

#### 4. 重置密码
* **接口地址**：`/api/v1/auth/reset-password`
* **请求方式**：`POST`
* **请求参数** (Body - JSON)：
    ```json
    {
      "email": "user@example.com",
      "new_password": "new_password123",
      "code": "123456"
    }
    ```
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "密码重置成功",
      "data": null
    }
    ```

### 二、 运势业务模块 (Fortune)

*(注意：以下接口需要在 HTTP 请求头 Header 中携带 `Authorization: Bearer <Token>` 进行 JWT 鉴权)*

#### 5. 获取今日运势
对应功能 4.1。后端逻辑：查库 -> 有则直接返回 -> 无则调用大模型 API -> 存库 -> 返回。

* **接口地址**：`/api/v1/fortune/today`
* **请求方式**：`GET`
* **请求头 (Header)**：`Authorization: Bearer <你的JWT Token>`
* **请求参数**：无
* **成功返回**：
    ```json
    {
      "code": 200,
      "message": "success",
      "data": {
        "date": "2026-04-19",
        "content": "今日星象绝佳，水星顺行带来沟通上的便利，适合开启新的项目。幸运色：海蓝色。"
      }
    }
    ```

### 💡 工程师的开发小贴士：

1.  **关于 HTTP 状态码 vs 业务状态码**：我在这里采用了 HTTP 状态码一律返回 `200 OK`，具体的业务错误（如“密码少于8位”、“邮箱已存在”）通过 JSON 里的 `code`（如 4001, 4002）和 `message` 来体现的设计。这在 Vue 前端使用 Axios 统一拦截响应时非常方便。
2.  **JWT 解析提取信息**：对于 `/fortune/today` 和 `/user/subscribe` 这类接口，前端不需要传 `user_id`。Golang 后端中间件（Middleware）在校验 JWT 成功后，应该直接从 Token payload 里解析出 `user_id` 并塞入上下文（如 Gin 框架的 `c.Set("user_id", uid)`），从而保证接口的安全，防止用户恶意篡改 `user_id` 获取他人运势。
