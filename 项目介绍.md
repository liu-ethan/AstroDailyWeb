【背景】：
假如你是一个软件开发工程师，你现在需要设计一个h5端的”测测你的每日运势“网页，前端使用Vue框架，后端使用Golang语言

【任务】：
请先完成后端部分：
目前所有通用组件以及实现好，并测试过
现在你需要根据项目介绍.md，接口文档.md，和项目中router已经写好的接口，来实现对应的逻辑
注意：每次实现接口后，都要写相关的测试代码，在同文件夹下，命名为xxx_test.go，来测试你实现的接口逻辑是否正确

【后端架构】
采用MVC架构，即Controller, Service, Mapper三层架构

【代码要求】
1. 请使用解耦思想，将不同的功能模块进行分层设计，确保代码的可维护性和可扩展性
2. 所有配置项统一写在一个配置文件中，路径是backend/config/config.yaml
3. 注释要求：请在函数级别添加注释，说明函数的功能、参数和返回值
4. AI Prompt设计：请设计一个通用的AI Prompt模板，按照标准Prompt Engineering的模板库来写，写在backend/internal/llm下面，方便后续调用大模型API生成内容

【功能介绍】：
页面1：用户注册
When 用户输入自己的邮箱，设置密码
Then 点击接收验证码（假设你已经有一个163邮箱的SMTP服务）
Then  输入验证码，点击确定
If 密码少于8位 Then 报错：密码不得少于8位
Else If 邮箱在数据库已存在 Then 报错：邮箱已存在
Else 用户注册成功，跳转3秒后跳转用户登录页面，密码和邮箱明文（不加密）存储到数据库中

页面2：用户登录
When 用户输入自己的邮箱，密码
Then 点击确定按钮
If [邮箱,密码] 不正确 Then 报错：邮箱或密码不正确
Else 用户登录成功，跳转<页面1>，生成JWT Token，有效时间30分钟，保存在Redis中，返回给前端

页面3：用户重置密码
When 用户输入自己的邮箱，需要重设的密码
Then 点击接收验证码（假设你已经有一个163邮箱的SMTP服务）
Then  输入验证码，点击确定
If 验证码不正确或邮箱不存在数据库中 Then 报错：请输入正确的邮箱或验证码
Else If 密码少于8位 Then 报错：密码不得少于8位
Else 用户更改密码成功，更新数据库，3秒后跳转用户登录页面

页面4：主页面
功能4.0 用户填写个人资料
When 用户进入主页面
Then 系统检查用户是否填写个人资料（姓名，出生日期，星座等）
If 用户未填写个人资料 Then 提示用户填写个人资料，跳转到个人资料填写页面，填写完user_profile表后，跳转回主页面
Else 显示用户个人资料和今日运势

功能4.1 点击按钮生成今日运势
When 用户点击“生成今日运势“按钮
Then 按钮变浅，变为不可点击，提示：请等待30秒钟，获取今天运势中....
If 数据库中有对应该用户该日的”运势“字段，则直接返回数据库
Else 先调用大模型API生成该用户今日的”运势“，存入数据库中，最后返回给用户

功能4.2 点击订阅该频道，每日8点自动发送今日运势邮件（适用于用户未订阅情况）
When 用户点击”订阅该频道，每日8点自动发送今日运势邮件“按钮
Then 按钮变浅，变为不可点击，提示：已订阅，以后每日8点将发送当天运势给你

功能4.3 点击”取消订阅该频道“（适用于用户已订阅情况）
When 用户点击”取消订阅该频道”按钮
Then 按钮变浅，变为不可点击，提示：已取消订阅，不再发送当日运势邮件给你
Then 取消定时任务

定时任务：每日8点扫描用户列表，发送每日运势
When 每日7:30分
Then 遍历整个数据库，挑选已订阅的用户，调用今日API，获取今天运势
Then 将运势存入数据库中
Then 使用SMTP服务发送邮件给用户
Then 删除该用户超过7日的“每日运势”数据（防止数据库太大）
