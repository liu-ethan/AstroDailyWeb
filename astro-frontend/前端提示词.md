# 前端提示词（H5：每日运势）

你是一个资深前端工程师，请基于已有后端实现与接口文档，完成 H5 端页面开发。项目结构为 Vue3（Vite）。请严格对齐接口功能，并确保视觉美观、移动端适配、交互友好。

## 目标与视觉风格（参考手机 QQ 原生 H5 风格）
- **整体基调**：移动端优先，采用类原生 App（Native）的交互体验。年轻、活力、清爽、大圆角。
- **色彩规范**：
  - **主色（Brand Color）**：使用经典的 QQ 蓝 `#12B7F5` 作为全局按钮、高亮文字和激活状态的颜色。
  - **背景色**：表单类页面（登录/注册）使用纯白背景 `#FFFFFF`；主页面使用浅灰背景 `#F5F6FA`。
  - **文字颜色**：主标题/正文 `#333333`，次要信息/占位符 `#999999`。
- **字体排版**：使用系统默认无衬线字体。大标题加粗，表单和列表字号保持在 `15px` - `17px`。
- **UI 核心元素**：
  - **大圆角**：所有主要操作按钮均采用胶囊状满圆角（`border-radius: 99px`）。
  - **输入框**：摒弃传统边框，采用浅灰底色（`#F5F6FA`）的圆角矩形输入框（`border-radius: 8px`），内部无边框。
  - **卡片（主页用）**：主页的内容区块采用纯白大圆角卡片（`border-radius: 16px`），不加阴影或仅加极轻微的阴影。
  - **全局布局**：最大宽度 `420px`，自适应居中。高度占满屏幕 `100vh`。

## 必须实现的页面与功能

### 页面1：注册
- **UI 布局**：纯白背景。顶部左侧带“返回”图标。页面上方为大号加粗标题“欢迎注册”。
- **表单字段**：邮箱、密码、验证码（全部采用浅灰底色圆角输入框）。
- **“发送验证码”**：作为输入框内部右侧的纯文本按钮，颜色为 QQ 蓝，点击后变灰并倒计时。
  - 调用 `POST /api/v1/auth/send-code`，body: `{ "email": "user@example.com", "business_type": 1 }`
- **“注册”按钮**：底部满宽（带左右边距）的胶囊形主色按钮。
  - 调用 `POST /api/v1/auth/register`，body: `{ "email": "...", "password": "...", "code": "..." }`
- **交互提示**：
  - 密码少于 8 位：提示“密码不得少于8位”
  - 邮箱已存在：提示“邮箱已存在”
  - 成功：提示“注册成功”，3 秒后跳转登录页

### 页面2：登录
- **UI 布局**：纯白背景。页面上方为大号加粗标题“账号登录”。
- **表单字段**：邮箱、密码。下方配有次要颜色的“忘记密码？”和“新用户注册”文字链接。
- **调用** `POST /api/v1/auth/login`：
  ```json
  { "email": "user@example.com", "password": "12345678" }
  ```
- **鉴权**：成功返回 token 后，前端需保存 token（建议 localStorage），用于后续鉴权 Header：`Authorization: Bearer <token>`。成功后跳转主页面。

### 页面3：重置密码
- **UI 布局**：纯白背景，带顶部返回导航栏。
- **表单字段**：邮箱、新密码、验证码。
- **接口调用**：
    - 发送验证码：`POST /api/v1/auth/send-code`，business_type = 2
    - 提交重置：`POST /api/v1/auth/reset-password`，body 同上。
- **成功提示**：提示“密码重置成功”，3 秒后跳转登录页。

### 页面4：主页面（个人资料 + 今日运势 + 订阅）
- **UI 布局**：**浅灰背景 `#F5F6FA`**。内容分为三大区块，每个区块是一个**纯白大圆角卡片（border-radius: 16px）**，卡片间留有合理间距。

#### 4.0 卡片一：个人资料
- 字段：出生日期、星座、性别、城市、职业。采用上下/左右排列的表单项。
- 获取资料：`GET /api/v1/user/profile`
- 保存资料：`PUT /api/v1/user/profile`
- 若未完善资料：引导用户填写资料，点击“保存”按钮（QQ蓝小胶囊按钮）更新。

#### 4.1 卡片二：今日运势
- 调用：`GET /api/v1/fortune/today`
- 若已生成：以优美的排版展示在卡片内。
- 若未生成：显示为空白状态，配有一个“生成今日运势”的 QQ 蓝胶囊按钮。点击后按钮置灰 + 局部 loading 菊花图，提示“获取今天运势中...”。后端会调用大模型生成并返回。

#### 4.2 卡片三：订阅服务
- 包含一段简短的说明文字和右侧的一个 Switch 开关（或胶囊按钮）。
- 订阅：`POST /api/v1/user/subscribe`（无 body）
- 取消：`POST /api/v1/user/unsubscribe`（无 body）
- 交互：切换开关/点击按钮时，请求接口，成功后弹出 Toast 提示：
    - 订阅成功：“已订阅，以后每日发送当天运势给你”
    - 取消成功：“已取消订阅，不再发送当日运势邮件给你”

## 全局交互与接口规范
- **统一返回处理**：
  ```json
  { "code": 200, "message": "success", "data": {} }
  ```
    - code != 200 视为错误，message 直接通过 Toast 展示。
- **轻提示（Toast）**：用于成功、报错提示。采用暗色半透明背景 `rgba(0,0,0,0.7)`，白色文字，屏幕居中显示，2-3 秒后自动消失，无需带图标。
- **按钮状态**：所有操作按钮必须支持 disabled（透明度 50% 且不可点击）和 loading（显示加载圈）状态。

## 工程与架构建议
- 封装 `api.ts`：使用 Axios 或 Fetch，统一管理 baseURL、请求拦截器（自动注入 Token）和响应拦截器（统一错误 Toast）。
- 封装状态管理：使用 Vue `ref` 或 `Pinia` / `useAuth` 组合式函数管理 token 与登录状态。
- 页面路由设计：`/register` | `/login` | `/reset` | `/home` (包含 profile)

## 最终交付要求
- 完整可运行的 Vue 3 单文件组件（`<script setup>` 语法）。
- 完整的交互逻辑（状态切换、加载防抖、路由跳转、错误处理）。
